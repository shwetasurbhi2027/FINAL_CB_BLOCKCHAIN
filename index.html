Why Blockchain?

CertiChain uses blockchain as a tamper-proof trust layer so certificate authenticity can be checked transparently.

What Improved Today?

The app now supports read-only verification without wallet connection, stronger UI, and a smoother demo flow.

Who Can Use It?

Colleges, event organizers, student societies, and recruiters can all issue or verify trusted certificates.

Session Insights Applied

Only proof goes on-chain

Certificates stay off-chain. The app stores only a SHA-256 hash so data remains lighter, cheaper, and privacy-friendly.

Read and write are different

Adding a certificate needs wallet approval, while verification can be done transparently as a read operation.

Trust comes from verifiability

CertiChain focuses on authenticity checks instead of raw storage, which makes blockchain use more meaningful.

Improved Project Flow

  1. User uploads a certificate file and the browser generates a SHA-256 hash locally.
  2. User can register that proof on-chain using addCertificate().
  3. Anyone can later paste the same hash and verify whether it is already registered on Sepolia.
  4. The app shows validity, issuer wallet, and the recorded timestamp.
